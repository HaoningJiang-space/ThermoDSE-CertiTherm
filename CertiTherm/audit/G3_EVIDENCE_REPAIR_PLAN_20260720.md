# G3 Evidence Repair Start Packet — 2026-07-20

## Round question

Can CertiTherm evaluate a content-bound workload/package matrix without
relabeling one aggregate power trace, reusing one thermal operator across
packages, or mistaking a larger uncertainty set for a uniform-power baseline?

Active gate: G3 breadth and systems cost. The corrected G2 query semantics at
commit `dc06d77a831a5d4c8b9a3eed31c8d0368acf9e4d` are the prerequisite.

## Scope

In scope:

- replace the mutable, on-the-fly G3 demonstration with a registered suite
  loader and replayable cross-candidate query runner;
- require independently content-bound workload power, package thermal
  operators, point-estimate power, and physical provenance;
- compare a true singleton point estimate against the registered spatial
  equivalence class using the same candidates, objectives, thermal limit, and
  backend;
- preserve and explicitly retract the 2026-07-20 `5/8` pilot result;
- add deterministic adversarial tests for workload relabeling, package cache
  aliasing, non-singleton point baselines, and candidate-level/query-level
  confusion.

Non-goals:

- no new physical result or numeric paper claim;
- no synthetic replacement for missing placed-power evidence;
- no claim that point/spatial disagreement is an error rate;
- no G4 acquisition or minimum-information claim;
- no on-the-fly mutation of a shared HotSpot configuration or ptrace.

## Mechanism and fairness contract

One registered suite contains every workload-family × package-regime stratum.
Each stratum is a complete architecture-selection query whose candidate pool
covers every registered architecture family. Each candidate binds:

1. a workload-specific placed-power observation and provenance digest;
2. a package-specific thermal response and thermal-config digest;
3. a point-estimate power vector that is inside the spatial domain;
4. a nonthermal objective and deterministic tie rank.

The point baseline fixes every component with `lower == upper == point_power`.
The spatial method uses the supplied observation equivalence class unchanged.
Both paths call `decide_architecture_query`; candidate-bound status alone is
never labeled an architecture flip.

## Baselines and metrics

This repair implements only the original point-estimate baseline and the exact
CertiTherm query. Sampled stress, component-box, and fixed-refinement baselines
remain required before G3 can close.

Permitted metrics are certification coverage, non-identifiable fraction,
point-commitment-not-identifiable count, reachable outcomes, query/runtime/RSS,
artifact size, and replay status. `error rate`, `false-safe rate`, and regret
remain unavailable until a registered physical outcome oracle supports them.

## Acceptance, kill, and rollback

Acceptance conditions:

- a full Cartesian suite is required and duplicate/missing strata fail closed;
- reused workload power digests across different workload families fail;
- reused response/config digests across different package regimes fail;
- every point vector directly replays inside its spatial domain;
- point observations are provably singleton;
- every point and spatial query emits a content-bound artifact whose fresh
  replay passes;
- targeted and full tests pass on a clean committed moe-server checkout.

Kill condition: if real workload-specific placed power or package-specific
thermal operators are unavailable, G3 remains `OPEN`; no synthetic row is
substituted.

Rollback: preserve `b7b08ee` as the invalidated pilot tip and implement only on
`round/g3-evidence-repair`. Never rewrite the shared history.
